# Metadata schema

This document defines the authoritative on-disk schema. **Per-paper YAML is the source
of truth.** `registry/papers.jsonl`, `registry/catalog.sqlite`, and everything under
`indexes/` are *derived* and can always be regenerated by `scripts/update_indexes.py`.

There are two YAML records per paper:

1. `library/<slug>/paper.yaml` — the canonical **paper-level** record (one per paper).
2. `library/<slug>/<version_key>/metadata.yaml` — one **version-level** record per version.

Versioning rule: a new arXiv version, a changed PDF SHA256, or changed venue/acceptance
metadata produces a **new version directory** (`v2`, `v3`, ...). PDFs and summaries are
**never overwritten** for a new version — they are colocated in their own `vN/` dir.

---

## Identifiers

- `canonical_key` — stable key for the paper. Derivation priority:
  1. `arxiv-<base_id>` (e.g. `arxiv-2501.12345`)
  2. `doi-<slugified-doi>`
  3. `openreview-<id>`
  4. `s2-<semantic_scholar_id>`
  5. `title-<12-hex-of-sha256(normalized_title)>`
- `slug` — filesystem-safe human-readable dir name: `<short-title>-<short-key-suffix>`.
- `version_key` — `v1`, `v2`, ... assigned sequentially per canonical paper.
- `arxiv_version` — the arXiv-reported version (`v1`, `v2`, ...), tracked independently
  of `version_key` (a paper's `v1` directory may hold arXiv `v3`).

## `paper.yaml` (paper-level, authoritative)

```yaml
canonical_key:        # str, required, unique across library
slug:                 # str, required, == directory name
title:                # str, required
authors: []           # list[str]
year:                 # int | null
primary_ids:
  doi:                # str | null
  arxiv_base_id:      # str | null  (e.g. "2501.12345", no version)
  openreview_id:      # str | null
  semantic_scholar_id:# str | null
  openalex_id:        # str | null
canonical_urls: []    # list[str]
topic_groups: []      # list[str], subset of config/interests.yaml topic_groups
tags: []              # list[str], ideally from config/tag_taxonomy.yaml
venue:
  name:               # str | null
  tier:               # A_star | strong_preprint_signal | workshop_or_minor | unknown | null
  status:             # accepted | rejected | under_review | preprint | unknown | null
  year:               # int | null
user:
  read_status: unread # unread | reading | read | abandoned
  triage_label:       # MUST_READ | READ_SOON | SKIM | TRACK_ONLY | SKIP | null
  triage_confidence:  # low | medium | high | null
  notes:              # str | null  (free-form; preserved across re-summarization)
milestone:            # null, or a curated-landmark block (set by /milestone-papers):
  # field:            #   field key from config/milestones.yaml (e.g. computer_use_agents)
  # era:              #   emergence | foundational | expansion | recent
  # significance:     #   one-line why-it-matters
knowledge_concepts: []# list[str] concept slugs in knowledge/ linked to this paper
                      # (bidirectional with knowledge/concepts/<slug>.md: related_papers)
versions:             # list, newest last; each mirrors the version dir
  - version_key:      # "v1"
    version_label:    # human label, e.g. "arXiv v2"
    source:           # arxiv | openreview | openalex | semantic_scholar | manual | pdf_url | local_pdf
    source_url:
    pdf_path:         # relative to papers/ root, e.g. library/<slug>/v1/paper.pdf
    summary_path:
    metadata_path:
    fetched_at:       # ISO8601
    source_updated_at:# ISO8601 | null
    arxiv_version:    # "v2" | null
    pdf_sha256:       # str | null
    pdf_bytes:        # int | null
    summary_status:   # none | triage_only | full | stale
    extraction_status:# none | partial | full | failed
    created_at:       # ISO8601
    updated_at:       # ISO8601
relations:
  duplicate_candidates: []  # list[canonical_key]
  related_papers: []        # list[canonical_key | url]
  supersedes: []            # list[canonical_key]
  superseded_by: []         # list[canonical_key]
```

## `metadata.yaml` (version-level, authoritative)

```yaml
canonical_key:
version_key:
title:
authors: []
abstract:               # str | "Not reported"
venue:
year:
source:
source_url:
pdf_url:
pdf_sha256:
fetched_at:
source_updated_at:
ids:
  doi:
  arxiv_id:             # with version, e.g. "2501.12345v2"
  arxiv_base_id:
  arxiv_version:
  openreview_id:
  semantic_scholar_id:
  openalex_id:
tags: []
topic_groups: []
scoring:
  score:                # float 0..100 | null
  label:                # triage label | null
  confidence:           # low | medium | high | null
  rationale:            # str
  matched_interests: [] # list[str]
  negative_signals: []  # list[str]
  subscores: {}         # dict[str,float] (per dimension, 0..1)
artifacts:
  pdf:                  # relative path | null
  extraction:          # relative path | null
  summary:             # relative path | null
  code_url:            # str | null
  project_url:         # str | null
  dataset_url:         # str | null
status:
  downloaded:          # bool
  extracted:           # bool
  summarized:          # bool
  triaged:             # bool
  user_read_status:    # unread | reading | read | abandoned
```

## Required fields (validated)

`validate_registry.py` enforces, per record:

- `paper.yaml`: `canonical_key`, `slug`, `title`, at least one `versions[*]`.
- `metadata.yaml`: `canonical_key`, `version_key`, `title`.
- `canonical_key` unique across all `paper.yaml`.
- `version_key` unique within a paper.
- If `status.downloaded` true → `artifacts.pdf` exists on disk and SHA matches `pdf_sha256`.
- If `status.summarized` true → `artifacts.summary` exists.
- No two version dirs share a `pdf_sha256` unless one lists the other in
  `relations.duplicate_candidates` or is marked superseded.

Use `"Not reported"` for genuinely absent source data and `null`/empty for not-yet-filled
fields. Never invent metadata to satisfy the schema.

## Knowledge base (`knowledge/`)

Global, paper-agnostic notes. Authoritative form: `knowledge/concepts/<slug>.md`
(markdown + YAML front-matter). `knowledge/{INDEX,BY_TAG,BY_PAPER}.md` are derived
(regenerate with `scripts/knowledge.py index`).

Concept front-matter:
```yaml
name:               # slug == filename stem
title:              # human title
aliases: []         # abbreviations / alt names
tags: []            # ideally from tag_taxonomy.yaml
related_papers: []  # canonical_keys — reciprocal with paper.yaml: knowledge_concepts
related_concepts: []# other concept slugs (reciprocal)
created:            # ISO8601
updated:            # ISO8601
```
Body sections: `## Intuition`, `## Details`, `## Q&A`, `## See also` (cross-link with
`[[concept-slug]]`). Concepts are **global**: one concept may link many papers, and a
paper may link many concepts. `scripts/knowledge.py validate` checks reciprocity and
that `related_papers` exist in the registry. Session logs live in `knowledge/sessions/`.
