PASS 2 Content

Motivation — the problem (NOT the novelty)

Computer-use agents must do three cognitively distinct things at once — long-horizon planning, action generation, and pixel-precise grounding — and prior work tends to push all three through a single model. The paper argues this monolithic setup fails on two fronts simultaneously (p.2, sec.1): models imprecisely ground GUI elements to pixel coordinates, and they struggle on long-horizon tasks amid pop-ups, background apps, and evolving observations. A stated structural reason is that fine-tuning a single model to specialize on one sub-skill tends to erode its broader capabilities (catastrophic forgetting; Yosinski 2014, Luo 2023, p.3), so a single generalist is bottlenecked by whichever sub-skill it is weakest at. The best agents still lag humans by roughly a 40% gap on OSWorld (p.1) — so the problem is real and unsolved. This subsection is problem-only: it does not yet say what Agent S2 does differently.

Novelty — the genuine delta ★

  • Delta in one sentence (survives deleting "we propose"): grounding should not be one monolithic skill — route each action to a specialist grounder (visual model, OCR for text spans, a spreadsheet/UNO API for cells), and re-plan the remaining subgoals after every subgoal using fresh observations rather than only after a failure.
  • Mechanistic reason each piece is needed:
    • Mixture-of-Grounding — visual grounding models miss fine-grained targets like the exact edge of a word or an arbitrarily-resized spreadsheet cell (p.5, sec.3.1). Those failure classes have better non-MLLM tools: OCR pinpoints character spans, and a programmatic table interface (UNO) sets cell values directly. So letting the Worker route to the right tool removes failure modes a single visual grounder cannot fix, and frees the Worker to focus on reasoning.
    • Proactive Hierarchical Planning — the initial observation is only partial and the screen evolves (pop-ups, noise). A reactive plan only changes on failure, so it cannot incorporate information learned from a successful subgoal. Proactive re-planning lets the Manager recontextualize the user query as the state is revealed, which is what enables backward correction and adaptive exploration (Fig.3, Fig.7).
  • Closest prior work & precise difference: Agent S (same group) already used hierarchical Manager/Worker + a knowledge base, but the Worker did its own grounding and planning was not re-run every subgoal; Agent S2 adds the grounding router and proactive re-planning. Modular planner+grounder pipelines (UGround, Aria-UI, OmniParser/NAVI) use a single grounder paired with an LLM planner; Agent S2 uses a mixture of grounders chosen per-action. Monolithic native agents (UI-TARS, Claude Computer Use, OpenAI CUA) fold all skills into one model; Agent S2 deliberately composes separate ones.
  • Motivation-vs-novelty check: passes — the contribution is not merely "monolithic agents fail" (that is the motivation) but a specific architecture (per-action grounding routing + per-subgoal re-planning) plus the empirical claim that composing individually-suboptimal specialists can beat the best monolithic model (p.2). Litmus: ablations verify the novelty (MoG on/off, PHP on/off, swapping grounders), not just confirm the motivation.
  • 30-second test: "Treat grounding as a routing problem to expert tools, and re-plan after every subgoal — a stack of cheap specialists beats one big generalist on computer use."

Core idea / method

Three components (sec.3). A Manager M decomposes the instruction into high-level subgoals. A Worker W executes the top subgoal by emitting atomic actions, each tagged with a language descriptor of its target location. Instead of grounding that target itself, the Worker acts as a gate and routes to one of the grounding experts: a visual expert (description → ⟨x,y⟩ pixel coordinate, screenshot-only, UI-TARS-72B-DPO in their setup), a textual expert (Tesseract OCR returning start/end coordinates of a word span given two boundary phrases), and a structural expert (UNO interface that programmatically writes spreadsheet cells from a {cell:value} dict). On the planning side, after each subgoal returns SUCCESS/FAILURE, the Manager regenerates the remaining subgoals from prior subgoals + latest observation + original instruction (Proactive Hierarchical Planning). The framework reuses the Agent S knowledge base (task/subgoal experience + web knowledge). The action space (Tab.8, appendix) is a function-call-style interface including highlight_text_span and set_cell_values that surface the textual/structural experts.

Hidden assumptions & failure modes

Assumes the Worker can reliably choose the right grounding expert (routing is itself an LLM decision and can misroute); assumes language descriptors are unambiguous enough for the visual grounder; the structural expert assumes a UNO-controllable app (LibreOffice/soffice), so it does not generalize to arbitrary spreadsheets. Proactive re-planning runs the Manager after every subgoal, adding LLM calls/latency/cost that are never quantified. Per the authors' own error analysis, planning is now the dominant failure (41.0%, Fig.8) — the framework moves the bottleneck rather than removing it. Performance still trails humans substantially (~40% gap, p.1).

Glossary / key terms

Computer-use agent
An autonomous agent that completes user tasks by directly operating a GUI (keyboard/mouse), usable across apps without app-specific APIs.
Grounding
Mapping a textual description of a UI element ("the OK button") to exact pixel coordinates (or a specific accessibility-tree node) so the agent can act on it. A core bottleneck for GUI agents.
Mixture-of-Grounding (MoG)
Agent S2's mechanism where the Worker, acting as a gating function (analogous to Mixture-of-Experts), routes each action's localization to a specialist grounder: visual (image→coordinate), textual (OCR for word spans), or structural (programmatic spreadsheet cells via UNO).
Generalist vs specialist model
Generalist = one broadly-capable model doing planning/execution/grounding; specialist = a model/tool tuned for one sub-skill. The paper's thesis: composing specialists (even individually suboptimal) can beat the best single generalist.
Proactive vs reactive planning
Reactive = keep a fixed plan and revise a subgoal only after it fails. Proactive (this paper) = re-generate the remaining subgoals after every completed subgoal using the latest observation, enabling recontextualization and backward correction.
Proactive Hierarchical Planning (PHP)
Two-level proactive planning: the Manager re-plans subgoals per subgoal and the Worker re-routes its grounder per action, adapting at multiple temporal scales.
Manager / Worker
Hierarchical roles: Manager generates high-level subgoals; Worker executes the current subgoal as atomic actions and routes their grounding.
Mixture-of-Experts (MoE)
Classic architecture (Jacobs et al., 1991) where a gating network sends each input to specialized expert sub-models; the conceptual inspiration for MoG (here the Worker is the gate).
OSWorld
Benchmark of 369 real desktop tasks on Ubuntu (OS, office, daily, professional, multi-app), evaluated with functional success scripts; main benchmark here, run at 15- and 50-step caps.
WindowsAgentArena (WAA)
Parallelizable Windows benchmark (154 tasks) extending OSWorld-style desktop control to Windows apps.
AndroidWorld
Dynamic Android benchmark (116 mobile UI tasks across 20 apps) used to test cross-platform generalization.
Accessibility (a11y) tree
A structured, machine-readable representation of on-screen UI elements; some baselines feed it to the agent. Agent S2 deliberately avoids it, using screenshots only.
UNO
Universal Network Objects: a programmatic interface to LibreOffice used by the structural grounding expert to set spreadsheet cell values directly.
Native / monolithic agent
A single model post-trained to do planning + grounding + execution end-to-end (e.g. UI-TARS, Claude Computer Use, OpenAI CUA); the foil for the compositional approach.
