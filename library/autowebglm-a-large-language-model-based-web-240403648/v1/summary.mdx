## Motivation
Most LLM web agents perform far below usable levels in real navigation because of three coupled difficulties: <Problem>HTML is verbose and complex, so models struggle to read pages accurately</Problem>; <Problem>there is no universal action space covering the operations needed across diverse websites</Problem>; and <Problem>the open-domain web makes tasks hard, and agents lack inference and self-checking, so once stuck in an erroneous loop they cannot recover</Problem>. Prior approaches were either deployment-impractical (e.g. <Related>WebAgent's 540B-scale Flan-U-PaLM</Related>) or query-inefficient (<Related>Mind2Web's MindAct uses 10+ model calls per single web operation</Related>), and state-of-the-art models cannot reliably auto-annotate training traces.

## Contribution (what's new)
The genuine delta is a co-designed *interface + training recipe* that makes a 6B model a competent agent. <Novelty>An HTML-simplification algorithm (the "HTML Pruner") compresses pages into a concise operable representation while preserving structure and content</Novelty>, paired with a <Novelty>self-consistent, fixed action space of 10 operations</Novelty> so the model emits one executable command per step instead of multi-round element selection. Training then bootstraps the model: <Novelty>self-sampling DPO that builds contrastive pairs from the model's own wrong vs golden operations (teaching it from its own mistakes), with an added SFT-loss term to stabilize DPO</Novelty>, followed by <Novelty>rejection-sampling finetuning (RFT) that self-plays in sandbox environments and keeps only environment-verified successful traces</Novelty>. The recipe must take this form because pure SFT mimics reasoning but ignores page state and prior actions (causing hallucinated operations), and complex tasks exceed what any LLM can auto-label, forcing the hybrid human-AI data construction.

## Research questions
Can a small (6B) open LLM be made into a web-navigation agent that is both *capable* (rivaling GPT-4-class agents) and *practically deployable* in real browser environments — and what interface design and training stages are required to get there?

## Methodology
Web browsing is framed as sequential decision-making over a state (HTML, URL, window position) and an action set (click, type, scroll, etc.). An interaction framework parses raw pages (HTML parser + OCR) into simplified HTML, marks operable elements, and executes the model's predicted command; the loop runs until `finish` or a max length.

<Pass title="Method details">
**Observation space (§3.2.1).** Four indicators per step: task description, simplified HTML, current location (scroll position + page size), and past operation records. Past actions are passed explicitly to stop the agent looping on failed operations.

**Action space (Tab.1).** Ten operations: `click(id)`, `hover(id)`, `select(id, option)`, `type_string(id, text, enter)`, `scroll_page(direction)`, `go(direction)`, `jump_to(url, newtab)`, `switch_tab(id)`, `user_input(message)`, `finish(answer)`.

**Three-stage training (§4.2, Fig.6).**
<Stepper steps={[
  {title: "Step 1 — Curriculum SFT", body: "Easy→hard: first read/operate pages (4.1.1 data), then plan/reason and decompose tasks (4.1.2 data), yielding the base agent π_SFT."},
  {title: "Step 2 — Reinforcement Learning (self-sampling DPO)", body: "K=20 samples per complex task; keep cases solved 1..K-1 times (skip all-right and all-wrong); pair model errors vs golden ops; train with DPO + SFT loss (β=0.15, SFT weight 0.8) → π_DPO. Contrastive set ≈13k pairs."},
  {title: "Step 3 — Rejection-Sampling Finetuning (RFT)", body: "Self-play in MiniWoB++ and WebArena sandboxes; keep only env-verified successful traces (MiniWoB++ ≈15k traces / 66k steps; WebArena 240 traces / 2k steps, 64 samples each) and finetune per-environment."},
]} />

The SFT objective is `L_SFT = -E[(x,y)~D] log π_θ(y|x)`; RL combines DPO and SFT losses as `L_total = α·L_DPO + L_SFT`.

**Benchmark — AutoWebBench (§4.3).** Bilingual (EN/ZH), split into in-domain and out-of-domain; 50 verified traces per split; metric is Step Success Rate (SSR), following Mind2Web's per-step evaluation.

## Key findings
- <Finding src="Tab.6">Complex-task data is the dominant lever; simple-task data alone barely helps</Finding>: on MiniWoB++ adding Stage 1 (simple) over the base train set moves 44.3→48.3, while adding Stage 2 (complex) jumps to 78.9; Stage 1+2 reaches 81.7. Training on complex data alone causes basic operational errors that the simple data fixes.
- <Finding src="Tab.6">Each training stage compounds, and RFT is decisive in-domain</Finding>: SFT 81.7 → +DPO 80.8 → +RFT 89.3 on MiniWoB++, and 8.3 → 8.5 → 18.2 on WebArena — RFT roughly doubles WebArena success via environment self-play.
- <Finding src="Tab.6">DPO gives a modest but consistent lift on offline tasks</Finding>: Mind2Web 56.7→59.5 and AutoWebBench 61.8→62.7, supporting the claim that learning from its own mistakes reduces hallucinated operations.
- <Finding src="Tab.7">Hallucination is the leading failure mode at 44%</Finding>, followed by poor graphical recognition (28%), misinterpretation of task context (20%), and pop-up interruption (8%) — i.e. errors are dominated by reasoning/perception, motivating the proposed multimodal and self-check future work.
- <Finding src="Tab.5">Prediction (model inference) dominates per-step latency</Finding>: it is the largest component of execution time (~47.9% of average step time), with page loading second (~41.6%), so the agent's bottleneck is the model, not the browser plumbing.
- <Finding src="§4.2.2">Naive DPO was unstable; adding an SFT-loss term stabilized RL training</Finding> without losing the model's natural-language and agent abilities — a practical recipe detail rather than just an architecture claim.

## Implications
Co-designing a compact observation/action interface with a self-bootstrapping training pipeline lets a 6B model match GPT-4-class browser agents and run as a deployable Chrome extension, shifting the cost frontier for web agents away from raw scale. It extends the <Related>WebGPT / WebGLM lineage from QA toward operational web automation</Related> and contributes <Related>AutoWebBench, the first bilingual (EN/ZH) step-level web-navigation benchmark</Related>. Worth chasing next: <FollowUp>multimodal (HTML + screenshot) inputs for maps, animations, and video where HTML-only falters</FollowUp>; <FollowUp>reasoning/self-check strategies beyond Chain-of-Thought to verify state and the effect of an intended operation</FollowUp>; and <FollowUp>porting the recipe to mobile, where smaller viewports and gesture actions change the action space</FollowUp>.

## Limitations
- <Caveat>HTML-only observation fails on advanced/visual web apps (maps, animations, video)</Caveat>; the authors explicitly propose multimodal input as the fix.
- <Caveat>Absolute real-world performance is still low — 18.2% on WebArena</Caveat>, far from reliable autonomy on realistic open-web tasks.
- <Caveat>RFT gains are domain-specific</Caveat>: improvements come from per-environment self-play in MiniWoB++/WebArena sandboxes, so generalization to arbitrary live sites is unverified, partly due to network-policy constraints on real environments.
- <Caveat>Evaluation leans on step-level SSR and small test splits</Caveat> (50 traces per AutoWebBench split), which may overstate end-to-end task completion.
- <Caveat>Robustness drops on unfamiliar sites with unusual logic, and real environments are unstable</Caveat> (network, pop-ups), with no self-check mechanism yet in place.

## Critiques and Questions
1. Step Success Rate rewards matching individual golden steps; how much does high SSR translate to full-task completion on Mind2Web, where a single mis-step can derail a trajectory? The headline benchmark choice may flatter the model relative to WebArena's end-to-end success.
2. Several baselines in Tab.2/Tab.3 are zero/few-shot while AutoWebGLM is finetuned on in-distribution data (including merged Mind2Web/MiniWoB++ train sets) — is the comparison apples-to-apples, and how much of the lead is contamination/distribution alignment vs genuine capability?
3. MiniWoB++/WebArena numbers come from RFT *finetuned per environment*; a single general agent's cross-environment number isn't reported, leaving open how much is one model vs several specialists.
4. Many results appear to be single runs without variance; the WebArena 8.3→8.5→18.2 progression hinges on small absolute counts, so confidence intervals would matter.

## Glossary
- <Term def="Compresses raw HTML into a concise representation that preserves structure and key content while pruning redundant elements, so an LLM can read the page within its context window">HTML simplification / HTML Pruner</Term>
- <Term def="Step Success Rate: fraction of individual operation steps the agent performs correctly, evaluated per-step as in Mind2Web">SSR</Term>
- <Term def="Direct Preference Optimization: trains a model to prefer a 'chosen' over a 'rejected' response directly from preference pairs, without a separate reward model">DPO</Term>
- <Term def="Rejection Sampling Finetuning: sample many trajectories, keep only the ones verified correct by the environment/reward, then finetune on those successful traces">RFT</Term>
- <Term def="Training that orders examples easy-to-hard, mimicking human learning, here: first page reading/operating, then planning/reasoning">curriculum learning</Term>
- <Term def="A prompt that feeds all steps and key HTML of a trace at once and asks the model (GPT-4) to emit each step's intent, producing more coherent chain-of-thought than step-by-step prompting">global thought chain</Term>
- <Term def="Bilingual (English/Chinese) step-level web-navigation benchmark introduced by this paper, split into in-domain and out-of-domain">AutoWebBench</Term>
