<TLDR>A vision-first web agent that drives live, real-world websites end-to-end by acting on marked-up screenshots with an off-the-shelf multimodal LLM, paired with a GPT-4V "judge" that scores open-ended trajectories automatically and a 643-task / 15-site benchmark.</TLDR>

<WhyItMatters>Foundational for my computer-use / web-agent track: it popularized the "screenshot + Set-of-Mark + ReAct on a real browser" recipe and the "VLM-as-judge for open-ended trajectories" evaluation pattern that later agent papers reuse.</WhyItMatters>

## Motivation
Prior web agents share two gaps. (1) They consume only <Term def="the textual tree (roles, labels, hierarchy) a browser exposes for a page">HTML / accessibility-tree</Term> text and run inside simulators or static page snapshots, so they <Problem>ignore that browsers *render* pages into visual layouts</Problem> — and verbose HTML is hard to manage and discards layout cues. (2) Benchmarks like Mind2Web <Problem>grade each action against one predefined "golden" trajectory</Problem>, which cannot credit the many valid ways to finish a task. Together these <Problem>limit applicability to the messy live web (ads, pop-ups, constant updates)</Problem> and make fair, scalable evaluation of open-ended end-to-end success hard.

## Contribution (what's new)
<Novelty>A generic VLM can complete tasks on *unmodified live websites* by acting on a screenshot whose interactive elements are overlaid with numbered boxes — with no object-detection module and no fine-tuned element selector — and such open-ended runs can be graded automatically by a second VLM watching the screenshot trajectory.</Novelty> Mechanistic why: rendered pages encode information visually, and numbering elements lets the model *refer* to a target by index (`Click [17]`) instead of solving pixel-coordinate grounding, so one model can both see and act; the same numbers let the judge re-derive what happened without a golden path. Closest prior work: <Related><Term def="Set-of-Mark: overlay numbered boxes on an image for visual grounding">SoM</Term> (reused, but drops the object detector — elements extracted rule-based), WebArena (not self-hosted — runs the open web), and concurrent <Term def="a concurrent GPT-4V web agent whose best variant uses a fine-tuned cross-encoder to rank elements">SeeAct</Term> (WebVoyager needs no extra modules)</Related>.

## Research questions
Can an off-the-shelf multimodal LLM complete real-world tasks end-to-end on live websites using only marked-up screenshots plus minimal element text? And can a VLM reliably stand in for a human when scoring such open-ended trajectories?

## Methodology
Given an instruction, WebVoyager drives a real browser (Selenium) in an observe→think→act loop. Each observation is the current screenshot with numbered Set-of-Mark boxes plus per-element text (type, inner text, `aria-label`); the model emits a <Term def="prompting style where the model writes a free-text Thought before each Action">ReAct</Term> Thought then one Action from a fixed set, terminating with `ANSWER`. The evaluator is a separate GPT-4V given the task, the agent's response, and the last k trajectory screenshots, returning binary SUCCESS / NOT SUCCESS.

## Main result
WebVoyager reaches <Finding src="Abstract; Tab.1, p.7">59.1% human-judged task success (GPT-4V backbone) vs 40.1% text-only and 30.8% GPT-4 (All Tools)</Finding>, and its GPT-4V auto-judge hits <Finding src="Tab.2, p.7">85.3% agreement with humans, Cohen's kappa = 0.70 — matching the human inter-annotator kappa of 0.70</Finding>. <Caveat>The headline 59.1% is human-judged and appears single-run; only the GPT-4V-judged numbers carry variance (the same agent auto-judges to 57.1% +/- 0.2%), so abstract 59.1% vs Table 1 57.1% reflect different judges, not different agents.</Caveat>

## Key findings
- <Finding src="Tab.1, p.7">A marked-up-screenshot VLM agent beats both text-only and tool-based baselines by large margins on live sites — 59.1% vs 40.1% (text-only) vs 30.8% (GPT-4 All Tools)</Finding>: the vision-first recipe wins overall, and also wins GAIA (Fig.5).
- <Finding src="Tab.2, p.7">GPT-4V is a reliable automatic judge for open-ended trajectories — 85.3% agreement and kappa 0.70 with humans, on par with the human inter-annotator kappa of 0.70</Finding>; agreement climbs with more screenshots (75.3% → 79.7% → 81.3% at k=1/2/3), so the full trajectory matters.
- <Finding src="§5.2; Tab.1, p.7">No object-detection or fine-tuned selection module is needed — the rule-based GPT-4V-Act labeler suffices and still beats the best SeeAct agent (30% vs 26%) on its own online set</Finding>: simplicity does not cost accuracy here.
- <Finding src="§5.3, p.8; Tab.1">Both modalities are needed for a generalist agent: vision lags on text-heavy sites (GitHub/ESPN/Wolfram), while text-only collapses on visually complex ones (Booking 2.3%, Google Flights 7.1%)</Finding> — argued from per-site splits, not a clean ablation.
- <Finding src="Tab.3, p.7">Cross-backbone judging exhibits self-evaluation bias — GPT-4o is lenient, GPT-4V strict, and Claude-3-Opus prefers its own outputs</Finding>, which confounds the GPT-4V / Claude / GPT-4o comparison.
- <Finding src="Tab.4, p.9">Failures are dominated by Navigation Stuck (44.4%), then Visual Grounding (24.8%), Hallucination (21.8%), and Prompt Misalignment (9.0%)</Finding>: running out of the 15-step budget, not raw perception, is the top failure mode.

## Implications
A simple, module-free recipe (off-the-shelf VLM + rule-based marking + ReAct on a live browser) became <Related>a de-facto baseline architecture for multimodal computer-use agents</Related>, and the validated VLM-as-judge protocol gave the field a scalable way to score non-deterministic, open-ended agents without golden trajectories. <FollowUp>Worth chasing: whether feeding HTML/accessibility text into the vision agent closes the text-heavy-site gap, and how a single neutral fixed judge would change the cross-backbone ranking.</FollowUp>

## Limitations
<Caveat>Live-web evaluation is inherently non-reproducible — numbers are a snapshot in time as sites, ads, and real-time data change.</Caveat>
<Caveat>The headline 59.1% (human-judged) appears single-run with no variance; abstract 59.1% vs Table 1 auto-eval 57.1% are easy to conflate.</Caveat>
<Caveat>Self-evaluation bias: each backbone's judge favors its own outputs (Tab.3), confounding the cross-backbone comparison (GPT-4o lenient, GPT-4V strict, Claude prefers itself).</Caveat>
<Caveat>Site selection excludes login/CAPTCHA sites, biasing toward agent-friendly tasks; reads poorly on text-heavy pages; no Drag, single tab, 15-step budget. Documented failures: Navigation Stuck 44.4%, Visual Grounding 24.8%, Hallucination 21.8%, Prompt Misalignment 9.0% (Tab.4).</Caveat>

## Critiques and Questions
1. The "both modalities needed" claim is argued from per-site splits, not from an ablation that feeds HTML text into the vision agent — the cleaner test would add accessibility text to WebVoyager and measure the lift.
2. Cross-backbone scores (Tab.3) are confounded by self-evaluation bias; a single fixed neutral judge across all backbones would make the GPT-4V / Claude / GPT-4o comparison fair.
3. Headline 59.1% is single-run human-judged — I'd want repeated human passes or at least variance before treating the GPT-4V-vs-baseline gap as robust.
4. Site selection avoids login/CAPTCHA; how much does success drop on the harder, gated web that real assistants must handle?

## Glossary
- <Term def="an autonomous program that reads a web page and issues actions to complete a natural-language instruction without human help">Web agent</Term> — the system class.
- <Term def="Large Multimodal Model: an LLM that also takes images; here GPT-4V, Claude 3 Opus, GPT-4o serve as the agent's brain">LMM</Term> — the backbone family.
- <Term def="overlay numbered bounding boxes on interactive elements so a vision model can reference targets by index instead of pixel coordinates">Set-of-Mark (SoM)</Term> — the marking technique.
- <Term def="a JavaScript tool that finds interactive elements by HTML type and draws numbered boxes; rule-based, no learned model">GPT-4V-Act</Term> — how the boxes are produced.
- <Term def="fraction of tasks judged successfully completed (binary), regardless of whether the steps were optimal">Task Success Rate</Term> — the primary metric.
- <Term def="agreement corrected for chance; 0.70 is 'substantial', matching the human inter-annotator level">Cohen's kappa</Term> — judge-reliability statistic.
- <Term def="an LLM creates new tasks from a few seed examples; used with human filtering to build the 643-task set">Self-instruct</Term> — benchmark construction.
- <Term def="a general-assistant benchmark; here only its 90 web-browsing Level-1/2 tasks are used">GAIA</Term> — secondary benchmark.
