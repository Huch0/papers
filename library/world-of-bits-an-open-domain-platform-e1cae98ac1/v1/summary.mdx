## Motivation
Before this work, agent benchmarks were mostly <Problem>synthetic games or narrow, closed domains</Problem>. The web is a vast, open-domain action space humans already use, but the field lacked a <Problem>standardized, scalable platform to train and evaluate agents that operate real web interfaces through the same input modalities a person uses</Problem>.

## Contribution (what's new)
The genuine delta is a framing plus reusable infrastructure: <Novelty>treat web interaction as an RL environment — render a (mini) web page, expose DOM and pixels as observations and keyboard/mouse events as actions, and define reward from task completion</Novelty>. The design must use **low-level, human-style input** (not a high-level API) so that the same agent could, in principle, operate any real site; the controlled MiniWoB suite makes this reproducible.

## Research questions
Can web interaction be cast as a standardized, reproducible RL benchmark over human-style input channels, and how far can agents of the era get on it relative to humans?

## Methodology
Frame each task as: observation = the page (DOM structure and/or rendered pixels), action = low-level mouse clicks/movement and keyboard events, reward = task completion. Provide a suite of small, self-contained UIs (MiniWoB) plus the broader vision of using live web pages. Agents are trained with reinforcement learning and imitation from demonstrations; reproducibility comes from the fixed task set.

<Pass title="Harness framing (context-grounded)">
- **Environment / workspace:** rendered web pages — the MiniWoB suite of small self-contained UIs, plus the broader "open-domain web" vision.
- **Observation interface:** the page — DOM structure and/or rendered pixels.
- **Action interface:** low-level human-style input — mouse clicks/movement and keyboard events (not a high-level API).
- **Planner / executor / verifier:** a single RL policy; the "verifier" is the task reward.
- **Evaluation harness:** per-task success/reward on MiniWoB.
- **Training harness:** reinforcement learning and imitation from demonstrations.
- **Vs later work:** deliberately low-level and reproducible; later milestones trade some control for realism (real sites, larger action vocabularies).
</Pass>

## Key findings
Field-role observations that are well-established for this paper; all quantitative specifics are **Not reported / verify against paper**:
- <Finding src="verify against paper">RL agents solved the easiest tasks but lagged humans badly on harder ones</Finding>: the gap, not a single score, is the takeaway — general web interaction is hard for era methods.
- <Finding src="verify against paper">The lasting contribution is the platform and framing, not one strong empirical result</Finding>: MiniWoB's value is as reusable infrastructure that many later papers built on.
- <Finding src="verify against paper">Low-level, human-modality action space makes credit assignment hard for RL</Finding>: directly motivated MiniWoB++'s workflow-guided exploration.
- <Finding src="verify against paper">The (observation = page, action = click/type, reward = completion) framing became the field standard</Finding>: it is why later web/computer-use benchmarks share a common shape.
