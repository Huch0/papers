Motivation

No existing VLM agent controls a real, arbitrary computer end to end through the GUI using continuous, freely-positioned mouse/keyboard commands (p.1, §1). Prior systems each miss a piece: CogAgent does GUI understanding/planning but lacks a complete plan→act→reflect loop; AppAgent (phones) has no planning phase and can only click pre-labeled UI elements, never arbitrary pixels; and most prior environments/datasets (Mind2Web, Seq2act, Screen2Words) assume HTML/UI metadata or are browser/Android-only (p.2-4), whereas mouse+keyboard on raw pixels is the universal interface.

Contribution (what’s new)

The genuine delta is a real-desktop environment over VNC where the agent sees only screenshots and emits typed JSON mouse/keyboard actions — no accessibility tree, no set-of-marks labeling, so it must localize pixels itself (p.3, §3.1). On top of it: an explicit plan→act→reflect control loop where the VLM itself acts as the in-loop verifier deciding continue/retry/reformulate (p.3, §3.2), and the CC-Score, an alignment-based metric that scores a click by whether it lands in an annotated feasible bounding box rather than at an exact pixel (§4). The pixel-self-localization design is forced by removing element labels: the model cannot lean on a DOM, so localization must be learned and measured directly.

Research questions

Can a VLM agent control a real OS purely from screenshots and raw mouse/keyboard actions across multi-step tasks, and can lightweight fine-tuning close the click-precision gap to GPT-4V?

Methodology

Given a task prompt and a screenshot, the agent loops: plan subtasks → act (emit low-level mouse/keyboard JSON dispatched over VNC) → reflect on the after-screen and decide continue/retry/reformulate. A human-corrected dataset of real on-machine traces trains a CogAgent-Chat fine-tune, with extra localization data to sharpen clicking.

Method details
  1. Plan — From the current screenshot + task prompt, emit a JSON list of PlanActions (subtasks). ~60% of tasks need 3–5 plan steps; avg 4, max 13 (Table 5, p.14).
  2. Act — Per subtask, emit typed JSON actions parsed into VNC device events. Action space (Table 1, p.4): Mouse Move/Click/Double-Click/Scroll/Drag with button + integer pixel position; Keyboard Press (keysymdef) and Text; Wait. Coordinates are absolute pixels from top-left.
  3. Reflect — Consume the after-screen and output EvaluateSubTaskAction = success / need_retry / need_reformulate plus free-text advice fed into the next prompt (motivated by Kolb's experiential-learning cycle, p.2).

Dataset construction (§4, Appendix C): interactive annotation where GPT-4V drafts a response, a human corrects it to a “golden” label, and actions are actually executed on a real Linux/Windows machine (Fig.3); chosen/reject pairs are kept for future RLHF. Coverage spans office, booking, info retrieval, card games, entertainment, programming, and system operations.

Training (§5.2, Table 4): supervised fine-tune of CogAgent-Chat. To boost localization, COCO and Widget Captions are reformulated into click/drag tasks (click object center; drag a box) and Mind2Web sessions are converted into the ScreenAgent schema (CLICK/SELECT→click center; TYPE→click+text). Data are mixed across 4 phases with rising ScreenAgent-data share (20%→70%). Hyper-parameters (Table 6): 6258 steps, LR 1e-5 cosine, batch 8, weight decay 0.05.

Main result

On the 70-session / 898-screenshot test set, the fine-tuned model is comparable to GPT-4V overall (CC-Score 0.61 vs 0.63) yet dramatically better at click localization (Mouse Position accuracy 0.51 vs 0.12) (Tab.3, p.5). Key findings

  • Localization fine-tuning is the headline win: Mouse Position accuracy jumps from CogAgent’s 0.02 / GPT-4V’s 0.12 to ScreenAgent’s 0.51 (Tab.3, p.5) — the paper notes GPT-4V often refuses to emit precise coordinates (p.6).
  • ScreenAgent is clearly weaker at planning — Plan BLEU 0.31 vs GPT-4V 0.47 (Tab.3, p.5): the small base model lacks GPT-4V’s common-sense decomposition, so the system trades planning quality for click precision.
  • Raw CogAgent-VQA/Chat almost never emit valid JSON function calls (near-zero), only partly recovered with a GPT-3.5 parsing crutch; fine-tuning lifts ScreenAgent to GPT-4V-level format compliance (Mouse-Position key-presence 0.91 vs 0.85) (Tab.2, p.5): instruction-format adherence is a learned skill, not free.
  • Self-verification (Reflecting) is unreliable for every model — Situation-Assessment F1 ≤ 0.60 (GPT-4V 0.60, ScreenAgent 0.52) (Tab.3, p.5), so “human intervention is still necessary” (p.6) and the loop is not truly autonomous.
  • The dataset is small but structured: 39 sub-task categories across 6 themes, avg 1.5 actions per sub-task; action mix is Evaluate 29.24% / Plan 28.68% / Mouse 24.67% / Keyboard 15.29% / Wait 2.12% (§4, p.4) (Fig.4/5a) — reflection/planning dominate the trace.
  • Qualitatively, ScreenAgent recovers from a failed action via reflection; LLaVA repeatedly clicks bottom-left and CogAgent often fails to emit coordinates (§5.3, p.6) (Figs.7-8).

Implications

Removing the accessibility tree and scoring clicks by region (not exact pixel) is a reusable harness/metric template. Lineage: built on CogAgent (Hong et al., 2023), contrasted with AppAgent’s labeled-element clicking, and reuses Mind2Web (Deng et al., 2023) as converted training data; environment lineage runs through MiniWoB++, WebShop, Seq2act, META-GUI. Compare its CC-Score (trace similarity) against later real-OS computer-use agents that report end-to-end task success, and check the repo for the VNC env wrapper, action parser, and CC-Score implementation (https://github.com/niuzaisheng/ScreenAgent).

Limitations

  • Single-frame input only — no video / multi-frame; language capability is bounded by the base LM, and even GPT-4V handles non-English on-screen text poorly (author-stated, §6, p.7).
  • Tiny eval (70 sessions / 898 screenshots), single run, no confidence intervals or significance testing.
  • CC-Score measures similarity to a human trace, not task completion; no end-to-end success rate is reported as a single number; positioning tolerance depends on annotator-drawn boxes.
  • The “RL environment” framing is nominal — no RL training and no concrete reward model; chosen/reject pairs are only proposed for future RLHF.
  • No safety/sandboxing/action-gating layer; only an ethical statement.

Critiques and Questions

  1. Table 3 scores are computed only over successfully matched/parsed actions, partly conditioning out the format-failure problem — what is the true end-to-end full-task success rate on the real machine?
  2. Baselines aren’t apples-to-apples: LLaVA-1.5 is capped at 336px input and CogAgent needs a GPT-3.5 crutch to parse, so format-compliance comparisons are skewed.
  3. No ablation isolates the contribution of each curriculum phase or of the reflecting phase on full-task outcomes.
Why does CC-Score report a high Mouse Position accuracy (0.51) for ScreenAgent even though it never matches exact pixels?
CC-Score counts a click as correct if it lands inside an annotated feasible bounding box, not at an exact coordinate — region membership, so reasonable-but-imprecise clicks still score.
