A fine-tuned LLaVA-based smartphone GUI agent that pairs a multi-source environment representation (screenshot + OCR text layouts + action history) with a top-down decomposed action format, reaching reported SOTA step-level action accuracy of 79.05% on AITW and 88.27% on META-GUI.

A representative point in the trainable open-source MLLM GUI-agent line (vs GPT-4V-prompting agents), directly relevant to Android device-control harness design. Its load-bearing claim — that cheap OCR layout text beats fancier vision encoders for GUI grounding — is worth knowing before building any single-screen GUI agent.

Motivation

GUI agents need what the paper calls “comprehensive cognition” — exhaustive perception plus reliable action response — and prior work falls short on both. Agents built by prompting proprietary (M)LLMs (GPT-4V, ChatGPT) suffer a domain gap between GUI commands and natural language: a semantically valid generation like “open the address book” is not an executable {action: click, touch_point: ...} command (§1). Vision encoders are low-resolution (e.g. 224×224) and pretrained for high-level captioning, so small but semantically loaded icons (a magnifier = “search”) are missed, and a finite input window forces a visual/textual length trade-off (§1–§2).

Contribution (what’s new)

Two design choices layered on a stock LLaVA backbone. Comprehensive Environment Perception (CEP): perceive the screen twice — holistically via the screenshot through CLIP, and fine-grained via OCR text layouts listed as (item name, coordinate), fused in the prompt with the goal and the last h actions (§3.3). Conditional Action Prediction (CAP): refactor the redundant AITW JSON command into a top-down natural-language line — predict the action TYPE first, then the TARGET conditioned on type, splitting the ambiguous dual_point into scroll / click / tap (§3.4, Tab.1). The design must take this form because the baseline fails on grounding: direct coordinate regression from a low-res image is hard, so OCR layout text supplies the missing high-resolution detail and CAP turns one hard prediction into two easier conditional ones.

Research questions

Can a small, trainable open-source MLLM reach or beat API-prompting and prior trained GUI agents by (1) enriching environment perception with OCR layouts + action history, and (2) decomposing action prediction — and which of those input channels actually carries the gain?

Methodology

Full supervised fine-tuning of LLaVA on recorded GUI episodes, doing single-step next-action prediction. Each step the model sees the current screenshot, OCR layouts, up to 8 prior actions, and the goal; it emits one CAP-formatted action line that is parsed back to JSON for scoring.

Method details
  1. Backbone — LLaVA = LLaMA-2-chat-7B + CLIP ViT-L/14 + one-layer linear projector mapping image features into the text embedding space; concatenate image+text embeddings, decode, cross-entropy loss (Eq.2, p.3).
  2. CEP input — Xtext = PROMPT(goal g, OCR layouts L, last h actions); Ximage = raw screenshot. h=8 full actions in the main runs (Appendix A). Layout line example: ICON_SETTINGS: [0.1783, 0.8701] (§3.3).
  3. CAP output — Rewrite each ground-truth action into a top-down natural-language line; split dual_point into scroll (endpoints far apart), click (tap inside a bounding box, with item name), tap (no matching box) (Tab.1).
  4. Normalization — Snap click targets to the OCR bounding-box centroid; snap scrolls to four cardinal swipes (§3.4).
  5. Train/eval — Per-subset (separate) and joint (unified) on AITW; eval step-level action accuracy (all JSON fields must match). Max input 2048; lr 2e-5; `{8,10,12}` epochs; batch `{12,16}`/device; 4x A800 (Appendix A).

Evaluation follows Rawles et al. 2023: a coordinate is correct if it lands in the same bounding box or within 14% screen distance; scroll correct if main direction matches; typed text correct if the label is contained in the prediction. META-GUI adds F1 (input text) and BLEU (utterances). This is offline, single-step, label-match against logged episodes — not closed-loop device control.

Main result

CoCo-Agent unified reaches 79.05% overall AITW action accuracy, topping CogAgent (76.88, the prior trained SOTA) and Auto-UI unified (74.27) (Tab.3, p.6); the unified model also beats its own separate training (77.82). On the conversational benchmark, CoCo-Agent hits 88.27% action accuracy on META-GUI vs m-BASH 82.74 (prior SOTA) — roughly a 12-point gain, with item accuracy jumping to 91.72 from 85.90 (Tab.4, p.6).

Key findings

  • OCR layouts (+5.82%) and action history (+5.63%) are the dominant contributors; CAP alone adds only +0.66% (Tab.5, p.7): the perception channels, not the action refactoring, carry the gain (General: 57.81 → 58.47 with CAP → 64.29 with layouts → 69.92 with 8 full actions).
  • OCR layout text beats heavier vision modules: CoCo-Agent (CLIP + Layout) = 71.0 on General, above CogAgent’s high-res cross-module (65.4) and Auto-UI (65.9) (Tab.7, p.7): the paper’s most transferable insight — straightforward textual layouts work as a better “high-resolution module” than fancier encoders.
  • A replacement probe confirms layouts/history dominate: corrupting layouts drops action accuracy 70.96 → 57.55 and history → 58.45, but corrupting the image only → 59.69 (Tab.6, p.7) because layouts compensate for the lost image detail.
  • Multi-step planning is weak: predicting the next action is 70.96%, but predicting 3 actions ahead with no intermediate states collapses to 37.74% (Tab.8, p.7): the agent learns reactive patterns, not look-ahead planning.
  • The metric underestimates the agent: of the first 500 General actions, 118 (23.6%) differ from the label but only 54 (10.8%) strictly contradict the episode (§5.4.2, p.8) — the rest stay goal-consistent (e.g. press-enter vs click-magnifier both submit a search).
  • “Comprehensive” does not extend to dialogue: META-GUI utterance BLEU is 65.90, below plain LLaVA’s 67.24 (Tab.4, p.6): the gains are in action grounding, not conversational generation.

Implications

For harness design, a good accessibility/OCR text layer can outweigh a bigger vision encoder for single-screen GUI grounding — a concrete lesson when building an observation interface. The work extends the Auto-UI (Zhang & Zhang 2023) trainable-MLLM, screenshot-driven AITW line and directly contests CogAgent’s dedicated high-resolution cross-attention vision module by arguing OCR text replaces it more cheaply. Whether OCR-layout-as-high-res-module holds for live, closed-loop device control rather than offline single-step prediction, and how to fix the multi-step planning collapse (37.74% at 3-next) needed for full-episode execution, are the threads worth chasing.

Limitations

  • Author-stated: large data/training cost vs zero-shot methods; future-action (multi-step) prediction remains poor; the ultimate goal of full-episode execution on a simulated/real device is not reached (Limitations, p.9).
  • Evaluation is purely offline, single-step, label-match against recorded episodes — never a full episode on a device — with no seeds/variance/significance, so the 79.05 vs 76.88 margin has no error bars.
  • OCR is an unanalyzed hidden pipeline dependency (engine/version unnamed); its errors are never quantified.
  • The “API safety/privacy” advantage is asserted in the Ethics Statement, not measured by any experiment (p.9).
  • A minor internal inconsistency: Tab.3’s top half lists GoogleApps 76.45 / WebShopping 75.00 while the parameter-breakdown rows show 75.30 / 76.10.

Critiques and Questions

  1. If CAP contributes only +0.66% (Tab.5), is the “conditional action prediction” framing doing real work, or is the whole win essentially “feed the model OCR text and recent actions”? The headline framing emphasizes both novelties equally; the ablation does not.
  2. Single is the one subset where CoCo-Agent does NOT top all baselines — CogAgent reaches 93.49 vs 91.41 (Tab.3). Single-step segmented sub-goals are exactly where a high-res vision module should help, which slightly undercuts the universal “layouts beat vision” claim.
  3. With no variance reported, how robust is the ~2-point lead over CogAgent across seeds?

Personal notes

The strongest takeaway for harness work: a good accessibility/OCR text layer can outweigh a bigger vision encoder for single-screen GUI grounding — relevant when deciding how to build the observation interface. Watch the offline-vs-online gap: 79.05% step accuracy is NOT episode success, and 3-next drops to 37.74%, so do not read these numbers as end-to-end task completion. Follow-ups to add/compare: Auto-UI, CogAgent, AITW (Rawles 2023), META-GUI (Sun 2022), MM-Navigator. Check the released code (github.com/xbmxb/CoCo-Agent) for the OCR engine, training scripts, and seeds.
