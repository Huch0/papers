AssistantBench is a 214-task benchmark of realistic, time-consuming open-web tasks on which every current system scores below 26% accuracy, alongside SeePlanAct (SPA), a planning- and memory-augmented web agent that roughly doubles the answer rate of prior agents.

Directly relevant to the computer-use / web-agent track: it stresses agents on the whole web (multi-site, multi-hop, no sandbox) rather than a single site, exposing that open-web navigation is the dominant bottleneck for today’s agents.

Motivation

Realistic information-seeking tasks — monitoring a real-estate market, finding a nearby early-bird fitness class — take a human several minutes of browsing across multiple sites, yet closed-book LMs rely on parametric knowledge and hallucinate while retrieval-augmented LMs are capped by retrieval quality and can be hurt by irrelevant evidence. Existing agent benchmarks test only single-website interaction or operate inside a sandbox, so they do not measure whether agents can plan and reason over the entire open web.

Contribution (what’s new)

Two deltas. First, a benchmark of realistic, time-consuming, automatically verifiable tasks that require autonomous open-web navigation across many real sites — the design forces multi-site browsing precisely because single-site/sandbox benchmarks let agents succeed without true web navigation. Second, SeePlanAct (SPA): SeeAct extended with an explicit planning/re-planning component and a memory buffer, plus open-web actions (go back, navigate to URL, query a search engine), because the baseline SeeAct lacks state to carry information across the long trajectories these tasks demand.

Research questions

Can language agents (closed-book LMs, retrieval-augmented LMs, and web agents) solve realistic, time-consuming, multi-site web tasks? Where do they fail, and does adding planning + memory to a SOTA agent help?

Methodology

A three-stage human data-collection pipeline builds the benchmark; tasks have closed-form answers scored by a type-aware automatic metric. The agent SPA augments SeeAct with planning and memory and is compared against closed-book, retrieval-augmented, and ensemble baselines.

Method details
  1. Seed tasks — 18 trusted participants share recent time-consuming web tasks they actually performed; each manually reviewed/tweaked (e.g. add a date constraint) with gold answer, verification URLs, and an explanation → 72 tasks.
  2. Crowd expansion — Crowdworkers use seed tasks as templates (vary actor, runtime, rating system, etc.) → +102 tasks, manually validated.
  3. Domain experts — 35 Prolific experts (biology, law, medicine, geography, visual arts, …) qualify by listing professional sites, then author time-consuming tasks over them → 42 expert tasks (incl. 2 from seed).
  4. Splits — 214 unique tasks total; most go to test; a 33-task dev set (12 seed + 21 crowd) for analysis. Answers span 525 webpages across 258 websites.
  5. Automatic evaluation — Answer types: string (word-level F1), number (order-of-magnitude metric, zero once a full order of magnitude off), and dictionary (per-key F1 of recall/precision). Also report answer rate (fraction not abstaining), precision (accuracy when answered), and exact match.

Difficulty is annotated by closed-book accuracy: easy = both GPT-4 and Claude-3.5-Sonnet CB-1S score ≥0.5; medium = one of them ≥0.5; hard = neither correct. Most tasks are hard.

SPA’s number-scoring metric: max{0, 1 - log(max(A,A')/min(A,A'))}. Agents run on GPT-4-Turbo and Claude-3.5-Sonnet, capped at 30 execution steps (agents “succeed quickly and fail slowly”).

Main result

On the test set with GPT-4-Turbo, no system exceeds ~25% accuracy; the best is the SPA→CB ensemble at 25.2% (falling back to one-shot closed-book when the agent abstains) (Tab.1). The SOTA web agent SeeAct reaches only 4.1% accuracy (15.5% answer rate); SPA more than doubles this to 11.1% accuracy and 35.9% answer rate with higher precision (30.9 vs 26.3) (Tab.1). With Claude-3.5-Sonnet, SPA→CB reaches 26.4% (Tab.7, §A.8).

Key findings

  • Closed-book LMs beat web agents and retrieval LMs on raw accuracy, mainly via higher answer rate, but at the cost of precision (Tab.1): CB-1S hits 22.2% accuracy by answering 89.5% of tasks, while exhibiting low precision and a tendency to hallucinate.
  • Agents “fail fast or slow”: SPA’s accuracy is near zero for very short (<5 steps) or very long (>15 actions) trajectories and peaks around ten steps (Fig.5) — and AssistantBench tasks demand long interactions, explaining the difficulty.
  • Expert tasks are harder for closed-book models but easier for SeeAct than general tasks (Tab.3): 70% of expert answers sit on a single URL (vs ~20% for general tasks), so they need less multi-site navigation.
  • For web agents, navigation errors dominate (36.7% for SPA, 63.6% for SeeAct), followed by grounding failures (~20%) (Tab.4); 80% (SPA) / 97% (SeeAct) of errors are simply not producing an answer.
  • Closed-book errors are mostly hallucinated answers (85%) or outdated answers (15%); retrieval-LM errors are dominated by retrieval failures (80%, split across tool-related 38.5%, irrelevant context 50%, partial info 11.5%) (§6.2).
  • Proprietary ChatGPT with web search + code interpreter also errs on >90% of dev tasks (§6.3), most often by over-relying on search results to produce a wrong answer.
  • On a 31-task FanOutQA subset (Wikipedia-only, multi-hop), SPA beats SeeAct by 22.5 points and is the only system to reach exact answers (3/31) (Tab.2), though closed-book CB-1S still leads on accuracy (40.9%).

Implications

AssistantBench reframes web-agent evaluation around open-web, multi-site navigation rather than single-site (Mind2Web, WebVoyager) or sandboxed (WebArena, VisualWebArena) settings, and is positioned closest to GAIA, but scoped purely to web navigation without audio/video tools. SPA continues the SeeAct multimodal-agent line by adding planning + memory. Whether open-weight agents (Llama-3.2, in-progress) can close the gap and training data / methods to fix open-web navigation, the dominant failure mode are flagged as the key next threads; extending to time-dependent tasks via reference-free evaluation models is another.

Limitations

  • Coverage: realistic but non-auto-verifiable, time-dependent tasks (e.g. live ticket availability) had to be excluded; tasks are limited to reachable participants.
  • Size: 214 tasks is smaller than some agent benchmarks — defended by appeal to small high-quality benchmarks (HumanEval 164, Bamboogle 125) and lower cost/footprint.
  • All systems use only GPT-4-Turbo and Claude-3.5-Sonnet, limiting reproducibility; open-source agents were still in progress.
  • ~56.5% of tasks can still change over a longer horizon (only 43.5% are fully static with date constraints, §6.4), so the benchmark will age.

Critiques and Questions

  1. The “best score” headline (25.2%) is an ensemble whose accuracy comes largely from the closed-book fallback (CB-1S alone is 22.2%); the standalone agent contribution is modest (SPA 11.1%). How much of the win is genuine web navigation vs. parametric recall plus a fallback?
  2. Precision when answering is high for agents (SPA 30.9%) but answer rate is low (35.9%) — is the real problem capability, or over-conservative abstention/grounding?
  3. With answers spread across 258 sites, how stable is the benchmark to link rot and content drift, given >56% of tasks are not strictly static?
Why does SeeAct find the expert task split easier than the general split, despite experts being domain specialists?
Because ~70% of expert-task answers can be found on a single URL (vs ~20% for general tasks), so they require far less open-web, multi-site navigation — the exact thing agents struggle with (Tab.3, §6.1).
